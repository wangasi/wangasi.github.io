数据库基础知识

数据库概念

  • 外键：为某个表中的一列，它包含另一个表的主键值，定义了两个表之间的关系。
  • 笛卡尔积：由没有联结条件的表关系返回的结果为笛卡尔积。检索出的行的数目将是第一个表中的行数乘以第二个表中的行数。
  • 联结的性能：Mysql在运行时关联指定的每个表以处理联结，这种处理可能是非常耗费资源的，因此不要联结不必要的表，联结的表越多，性能下降越厉害。
  • 自然联结： 排除多次出现，使每个列只返回一次。
  • 外部联结： 相关表中没有关联的行。
  • InnoDB是一个可靠的事务处理引擎，不支持全文本搜索。
  • MEMORY在功能等同于MyISAM,但由于数据存储在内存中，速度很快。
  • MyISAM是一个性能极高的引擎，支持全文本搜索，但不支持事务处理。
  • 视图的优点：
    1. 重用SQL的语句。
    2. 简化复杂的SQL操作，在编写查询后，可以方便的重用它而不必知道它的基本查询细节。
    3. 使用表的组成部分而不是整个表。
    4. 保护数据。可以给用户授予表的特定部分的访问权限而不是表的访问权限。
    5. 更改数据格式和表示。视图可返回与底层表的表示和格式不同的数据。
  • 如果MYSQL不能正确的确定被更新的基数据，则不允许更新。视图不能更新的情况：
    1. 分组(使用GROUP BY 和 HAVING)
    2. 联结
    3. 并
    4. 聚集函数(Min()、Count()、Sum()等)
    5. DISTINCT
    6. 导出(计算)列

数据库常用指令

  • 登录到DBMS mysql -u user_name -p -h localhost -P 3306
  • 使用表 USE crashcourse;
  • SHOW DATABASES; 查看数据库
  • SHOW TABLES; 获得数据库内所有表
  • SHOW COLUMNS FROM customers; 显示数据库内表列
  • DESCRIBE customers; 功能同上
  • SHOW STATUS; 显示广泛的服务器状态信息
  • SHOW CREATE DATABASE; 和 SHOW CREATE TABLE; 显示创建特定数据库或表
  • SHOW GRANTS; 显示授予用户的安全权限
  • SHOW ERRORS; 和 SHOW WARNINGS; 显示服务器错误或警告消息
  • INFORMATION_SCHEMA; 获得和过滤模式信息
  • SELECT * FROM customers; 检索所有列
  • SELECT DISTINCT vend_id FROM products; 过滤掉重复数据，只返回id不同的行
    备注： DISTINCT关键字应用于所有列而不仅是前置它的列
  • LIMIT 限制返回结果。
    1. SELECT prod_name FROM products LIMIT 5; 指示返回结果不多于5行
    2. SELECT prod_name FROM products LIMIT 3, 4; 返回从行3开始的4行 == LIMIT 4 OFFSET 3
      备注： 1）检索出来的第一行为0而不是1。2) 在行数不够时，返回能返回的行数。
  • ORDER BY对输出进行排序。
    1. SELECT prod_name FROM products ORDER BY prod_name;
    2. SELECT prod_id, prod_price, prod_name FROM products ORDER BY prod_price, prod_name; 先按价格，再按名称排序。
    3. DESC 降序排序。SELECT prod_name FROM products ORDER BY prod_name DESC;
    4. SELECT prod_id, prod_price, prod_name FROM products ORDER BY prod_price DESC, prod_name; 以降序排序产品，再对产品名排序。
      备注： DESC关键字只应用到直接位于其前面的列名。ASC升序，默认。
    5. SELECT prod_price FROM BY prod_price LIMIT 1; 找出一个列中最高或最低的值。
  • WHERE 搜索指定条件
    1. SELECT prod_name, prod_price FROM products WHERE prod_price=2.50; 只返回价格为2.50的行. —> 相等测试，检查一个列是否具有指定的值，以此过滤。
    2. SELECT prod_name, prod_price FROM products WHERE prod_price BETWEEN 5 AND 10;
    3. SELECT prod_name FROM products WHERE prod_price IS NULL;
    4. AND 和 OR 用来表示检索匹配满足所有条件的行 和 任一条件的行。
      SELECT prod_name FROM products WHERE vend_id = 1002 OR vend_id = 124;
    5. SELECT prod_name FROM products WHERE vend_id IN (1002,1003);
    6. SELECT prod_name FROM products WHERE vend_id NOT IN (1002,1003);
  • 通配符过滤,LIKE
    1. (%)通配符，表示任何字符出现任意次数。
      SELECT prod_id, prod_name FROM products WHERE prod_name LIKE ‘jet%’; 搜索jet开头的词。
    2. (_)通配符，只匹配单个字符。
    3. 通配符搜索的处理比上述搜索花时间多。
    4. LIKE 匹配整个列，如果被匹配的文本仅在列值中出现，like将不会匹配。
  • 正则表达式匹配
    1. SELECT prod_name FROM products WHERE prod_name REGEXP ‘.000’; 匹配含有000的值
  • concat()拼接串。SELECT Concat(vend_name, ‘(‘, vend_country,’)’) FROM vendors ORDER BY wend_name;
  • RTrim()去掉值右边的所有空格。LTrim()去掉值左边的空格，Trim()去掉串两边的空格。
  • AS 一个字段或列的别名。
  • 数据处理函数。
    1.upper()，将字母转换为大写。
    2.SOUNDEX(), 将任何文本转换为描述其语音表示的字母数字模式的算法
  • WHERE过滤指定是行，HAVING过滤分组。

  • GROUP BY和ORDER BY的区别
    | ORDER BY | GROUP BY |
    | ——————— | ————————————————- |
    | 排序产生的输出任意列都可 | 分组行。但输出可能不是分组的顺序 |
    | 以使用(甚至非选择的列也 | 只可能使用选择列或表达式列，而且必须使用每个选择列表达式 |
    | 可以使用)不一定需要 | 如果与聚集函数一起使用列(或表达式)，则必须使用 |

    group by子句用来按订单号分组数据，以便sum(*)函数能够返回总计订单价格。having子句过滤数据，使得只返回总计订单价格大于等于50的订单。

  • select子句顺序：select > from > where > group by > having > order by > limit
  • fulltext：在创建表时启用全文本搜索，给出被索引列的一个逗号分隔的列表。

  • Match()和Against()执行全文本搜索，Match()指定被搜索的列，Against()指定要使用的搜索表达式。

  • ALTER TABLE更改表结构。

    1. ALTER TABLE vendors ADD vend_phone CHAR(20);
    2. ALTER TABLE vendors DROP COLUMN vend_phone;
    3. ALTER TABLE orderitems
      ADD CONSTRAINT fk_orderitems_orders
      FOREIGN KEY (order_num) REFERENCES orders (order_num);
