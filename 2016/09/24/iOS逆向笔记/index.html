发表于 2016-09-24 | 分类于 iOS逆向 |

逆向基础知识

1、文件目录
1）操作系统目录（多用于底层）
a. /: 根目录
b. /bin 存放提供用户级基础功能的二进制文件，如ls.ps等。
c. /boot 存放能使系统成功启动的所有文件。
d. /dev 存放BSD设备文件。每个文件代表系统的一个块设备或字符设备。
e. /sbin 存放提供系统级基础功能的二进制文件。

f. /etc 存放系统脚本及配置文件，如password、hosts等。
g. /lib 存放系统库文件、内核模块及设备驱动等。
h. /mnt 存放临时的文件系统挂载点。
i. /private 存放两个目录，分别是/private/etc和/private/var。
j. /tmp 临时目录。
k. /usr 包含了大多数用户工具和程序。/usr/bin包含那些/bin和/sbin中未出现的基础功能，如nm,killall等；
、usr/include包含所有的标准C头文件；/usr/lib存放库文件。
l. /var 存放经常更改的文件，如日志、用户数据、临时文件等。/var/mobile和/var/root分别存放了mobile用户
和root用户的文件，是重点关注的目录。/var/mobile/Library/SMS下存放短信数据库；/var/mobile/Library/Mail
下存放邮件数据
2）文件目录
a. /Applications/ 目录存放系统App和从Cydia下载的App。
b. /Developer 如果一台设备连接Xcode后被指定为调试用机，Xcode就会在iOS中生成这个目录。其中
会含有一些调试需要的工具和数据。
c. /Library 存放一些提供系统支持的数据。/Library/MobileSubstrate下存放了所有基于CydiaSubstrate的插件。
d. /System/Library 存放大量系统组件。/System/Library/Frameworks和/System/Library/PrivateFrameworks
存放iOS中的各种framework.
e. /User 用户目录，实际指向/var/mobile。
f. /var/mobile/containers/ 目录存放的则是StoreApp。
g. /var/mobile/containers/Data App对应的Documents目录。/var/mobile/Media/DCIM下存放照片；
/var/mobile/Media/Recordings下存放录音文件

2、daemon 守护进程
1）daemon主要有一个可执行文件和一个plist文件组成。
2）iOS的根进程launchd会在开机时检查/System/Library/LaunchDaemons和/Library/LaunchDaemons下所有格式
符合规定的plist文件，然后启动对应daemon。

一、配置debugserver环境
1）debugserver只有在设备连接过一次Xcode后，并在window->devices菜单中添加次设备后，debugserver才会被
Xcode安装到iOS的/Developer/usr/bin/目录下。
2) 将debugserver从iOS拷贝到OSX的/user/iCrack/目录下。
3) lipo -thin arm64 ~/debugserver -output ~/debugserver
4) 下载http://iosre.com/ent.xml到OSX的/User/iCrack/目录下。
5）运行/opt/theos/bin/ldid -Sent.xml debugserver
6) scp ~/debugserver root@iOSIP:/usr/bin/debugserver
7) ssh root@iOSIP
8) chmod +x /usr/bin/debugserver
二、用debugserver启动或附加进程
1) debugserver IP:port /path/to/executable,如：
debugserver -x backboard *:1234 -a /Applications/MobileSMS.app/MobileSMS
上述代码会启动MobileSMS,并开启1234端口，等待任意IP地址的LLDB接入。
debugserver 192.168.1.103:1234 -a “Calculator” 附加进程
2) 启动lldb：在终端输入指令/Applications/Xcode.app/Contents/Developer/usr/bin/lldb
输入process connect connect://192.168.1.107:1234进行连接
3) image list -o -f 列举当前进程中的所有模块。
4）breakpoint
a. b function 在函数的起始位置设置断点，如：b NSLog
b. br s -a address 在地址处设置断点，如：br s -a 0xCCCCC
c. br s -a ‘ASLROffset+address’
d. c 让程序继续运行。
e. br dis 禁用断点，如：br disable 6，6为断点序号。
f. br en 启用断点
g. br del 删除断点
h. br com add 1 指定某个断点得到触发的时候，执行预先设置的指令。执行后，LLDB会要求我们设置一系列指令，以“DONE”结束。
i. ni(nexti) 执行此条指令,不进入函数体。stepi也是执行下一条机器码，但进入函数体。
j. print 打印值，如：p $r1
k. po 打印Object-c对象，如：po [$r0 class]
l. x 打印一个地址处存放的值，如：p/x $sp,以十六进制方式打印sp,是一个指针, x/10 $sp打印了这个指针指向的连续10个字的数据。
m. register write 命令用于给指定的寄存器赋值，从而对程序进行改动，观察程序的执行过程有什么变化。register write r0 1
n. 回车键，重复执行上一条指令。

arm64汇编指令：

  1. ldr r0, _start 从内存地址_start的地方把值读入。绝对地址
  2. adr r0, _start 取得_start的地址到r0，相对位置。
  3. 通过 B BL BLX BX 可以完成在当前指令向前或者向后32MB的地址空间的跳转
  4. b 是最简单的跳转指令。要注意的是，跳转指令的实际值不是绝对地址，而是相对地址——是相对当前PC值的一个偏移量，它的值由汇编器计算得出。
  5. bl 非常常用。它在跳转之前会在寄存器LR(R14)中保存PC的当前内容。

A64的新功能

  1. A64指令 提供同A32和T32指令类似的功能， 也提供如下新功能
    1. 一个清晰的，固定长度的指令集： 指令是32位宽度， 寄存器字段（fields）是 在固定位置的 连续的位，立即数 通常也是占用连续的位
    2. 更多通用寄存器(r0-r30), 每个寄存器长度扩展为64位。 通用寄存器 编码在5个位域
    3. 寄存器r31是一个特殊的寄存器：
      • Zero Register: 在大多数情况下，作为源寄存器使用时， r31读出来的值 是0; 作为目标寄存器使用时， 丢弃结果。 WZR(word zero rigiser)或者XZR(64位）
      • Stack Register: 当 用作load/store 的base register时， 或者 一些算术指令中， r31提供当前的stack pointer WSP或者 SP
    4. PC寄存器 不能直接读写。用来指明当前程序执行的位置
    5. 过程调用 链接寄存器(LR) 是 r30
    6. 异常处理 有个 ELR系统寄存器
    7. PC不是通用寄存器的一部分，不能通过寄存器号 来引用，所以不能作为 算术指令的源地址或者目标地址。
      也不能作为 内存存取的 基础(base), 索引(index)或者传输（transfer)寄存器。
      能够读取PC寄存器内容的指令是PC相对寻 指令(ADR, ADRP, 常量加载， 直接跳转等指令)
      跳转和链接指令，将PC保存到链接寄存器(BL和BLR)
      能够修改PC的唯一的方式，使用 隐式的控制流指令（条件跳转，无条件跳转，异常生成，异常返回）
      值得注意的是：PC 被这些指令 读出来时，其值 就是 该指令所在的地址，而不像 A32和T32中，要加上4或者8的偏移（历史上的原因，两级流水
      向后两条指令，如果是16位长度的thumb指令， 那就是PC+4;32位长的arm指令，就是PC+8)

内存数据存取

A32和T32中的LDM,STM, PUSH和POP指令，在A64中并不存在
又有一对新的指令LDP和 STP 用来从内存中存取数据到寄存器。
地址生成:

符号扩展一个21位的offset, 向左移动12位
PC的值的低12位 清零，然后把这两者相加， 结果写入到Xd寄存器
用来得到一块含有 lable的4KB对齐 内存区域的base地址 （也就是说lable所在的地址，一定落在这个4KB的内存区域里, 指令助记符里Page也就是这个意思)， 可用来寻址 +/- 4GB的范围

将有符号的21位的偏移，加上PC, 结果写入到Xd寄存器
可用来计算 +/- 1MB范围的 任意字节粒度的有效地址
